# 手工测试清单

## 功能测试

1. 首次进入页面，出现相机权限提示。
2. 拒绝权限后，页面显示“前往设置开启”入口。
3. 在系统设置开启权限后，返回页面可正常预览。
4. “实时增强叠层”开关可正常开关实时画面处理。
5. “原生变焦”滑杆可联动相机 zoom。
6. “数字变焦”滑杆可放大预览舞台。
7. 点击“拍照并增强”后展示原图与增强图。
8. 处理失败时出现错误提示与“重新处理”按钮。

## 按需注入/用时注入验证

1. 确认 `miniprogram/app.json` 含 `"lazyCodeLoading": "requiredComponents"`。
2. 首次进入页面时，不展示结果区域时不应提前渲染结果组件。
3. 首次拍照触发 `showResult` 后，先短暂显示占位组件，再切换为结果组件。
4. 页面二次展示结果区域时，不应再次出现明显组件注入卡顿。

## 画质测试

1. A4 文档：增强后文字边缘更清晰、背景更干净。
2. 书页弱光：文字可读性提升，噪点不过度放大。
3. 小字号屏幕文字：增强后仍保持可读，不出现明显锯齿。
4. 轻微反光/倾斜：页面不崩溃，结果稳定返回。

## 性能测试

1. 高性能机：实时模式下预览流畅，FPS 稳定。
2. 中档机：自动降采样后仍保持可操作性。
3. 低档机：可关闭实时增强，拍照增强流程可用。
4. 同时操作变焦与增强时，不出现明显卡死或闪退。

## 回归测试

1. 页面前后台切换后，可恢复相机与实时状态。
2. 多次连续拍照增强后，结果显示与按钮状态正常。
3. 关闭权限再进入页面，权限引导逻辑仍正确。
